# Provenance Guard 🛡️

A backend API that classifies whether submitted text was written by a human or
generated by AI. Built for a Love Island fan forum where readers deserve to know
whether the recap they're reading came from a real fan or a language model.

---

## Architecture Overview

A submitted post travels through the following path:

1. Creator sends `POST /submit` with their text and a creator ID
2. A unique `content_id` is generated for this submission
3. **Signal 1 (LLM)** reads the text and scores AI likelihood (0.0–1.0)
4. **Signal 2 (Stylometrics)** computes statistical properties and scores AI likelihood (0.0–1.0)
5. Both scores are combined into a single **confidence score** using a weighted average
6. The confidence score maps to one of three **transparency labels**
7. The decision is written to the **audit log**
8. The response is returned with `content_id`, `attribution`, `confidence`, and `label`

For appeals:
1. Creator sends `POST /appeal` with their `content_id` and reasoning
2. System updates the entry status to `"under_review"`
3. Appeal reasoning is logged alongside the original decision
4. Confirmation is returned to the creator

## Detection Signals

### Signal 1: LLM Classification (Groq)

**What it measures:** Semantic and stylistic coherence. The LLM reads the post
holistically and assesses whether it reads like a real fan wrote it — informal
language, personal opinions, slang, emotional reactions — or whether it reads
like AI output — overly structured, generic, balanced tone, no personality.

**Why I chose it:** A Love Island fan forum has a very distinct voice. The LLM
can pick up on whether a post actually sounds like someone who stayed up until
midnight watching the episode, or whether it sounds like a Wikipedia summary.

**What it misses:** A very articulate fan who writes formally might score high.
A prompt-injected input designed to mimic fan slang could fool it. The LLM has
no memory of prior posts so it can't detect patterns across submissions.

---

### Signal 2: Stylometric Heuristics (Pure Python)

**What it measures:** Statistical properties of the text that differ between
human and AI writing. Three specific metrics:

- **Sentence length variance:** AI sentences tend to be similar lengths. Human
  writing has more variation — short punchy sentences mixed with long ones.
- **Type-token ratio (TTR):** Vocabulary diversity. Unique words / total words.
  AI text reuses vocabulary in predictable patterns.
- **Punctuation density:** Human fan posts use `!`, `?`, `...` emotionally. AI
  text uses punctuation more sparingly and evenly.

**Why I chose it:** These metrics are genuinely independent from the LLM signal —
one is semantic, one is structural. That makes the combination more informative
than either alone.

**What it misses:** Short posts (under ~20 words) don't have enough text for
statistical signals to be meaningful. Non-native English speakers writing formally
might score high even though they're human.

---

## Confidence Scoring

Both signals are combined using a weighted average:
confidence = (0.6 × llm_score) + (0.4 × stylometric_score)

The LLM signal gets more weight because it captures semantic meaning, which is
more informative than surface statistics for opinion-based fan content.

**Score thresholds:**
- `0.00 – 0.40` → Likely human-written
- `0.41 – 0.65` → Uncertain
- `0.66 – 1.00` → Likely AI-generated

**Validation — two example submissions with noticeably different scores:**

| Input | LLM Score | Stylometric Score | Confidence | Attribution |
|-------|-----------|-------------------|------------|-------------|
| Formal academic-style post about monetary policy and asset prices | 1.0 | 0.653 | **0.861** | likely_ai |
| Casual review: "ok so i finally watched last nights episode and honestly? underwhelming..." | 0.2 | 0.519 | **0.328** | likely_human |

The scores vary meaningfully across the range — 0.861 vs 0.328 — confirming the
system is not producing constant or clustered outputs.

## Transparency Labels

All three label variants, exact text as displayed:

**High-confidence AI (confidence ≥ 0.66):**
⚠️ This post shows strong signs of AI generation.
The writing lacks the personality and spontaneity
we'd expect from a real Islander fan.

**Uncertain (confidence 0.41 – 0.65):**
🤔 We're not sure about this one. It could go
either way. Our system isn't confident enough
to make a call.

**High-confidence human (confidence ≤ 0.40):**
✅ This post reads as human-written — the chaos,
the slang, the passion. Verified fan energy.

## API Endpoints

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/submit` | Submit content for attribution analysis |
| POST | `/appeal` | Contest a classification decision |
| GET | `/log` | Return recent audit log entries |

**POST /submit** accepts:
```json
{
  "text": "string — the content to analyze",
  "creator_id": "string — identifier for the creator"
}
```

**POST /submit** returns:
```json
{
  "content_id": "string",
  "attribution": "likely_ai | uncertain | likely_human",
  "confidence": 0.78,
  "label": "string — transparency label text",
  "llm_score": 0.81,
  "stylometric_score": 0.72
}
```

## Rate Limiting

**Limits:** 10 requests per minute, 100 requests per day per IP address.

**Reasoning:** A real fan submitting their own recap or reaction post would
realistically submit a few times per day at most — 100/day is generous for
legitimate use. The 10/minute limit prevents scripted flooding while allowing
a user to quickly resubmit if they made an edit. These numbers reflect the
asymmetry of the platform: false positives hurt real creators, so the system
shouldn't be so restrictive that legitimate users are blocked.

**Verified behavior:** Sending 12 rapid requests produces 200s for the first
10 and 429 Too Many Requests for the remainder.

## Known Limitations

**Short posts are unreliable.**
The stylometric signal requires enough text to compute meaningful variance. A
post like "omg bestie WHAT 😭" is 4 words — there's no sentence length variance
to measure, no vocabulary diversity signal, and minimal punctuation pattern. For
posts under ~20 words, the stylometric score defaults to 0.5 (neutral), which
means the system relies entirely on the LLM signal. Short posts should be treated
as low-confidence classifications regardless of what the score says.

**Formal human writing gets misclassified.**
A non-native English speaker who writes in a structured, careful style — or a fan
who happens to write analytically — will trigger both signals toward AI. The LLM
reads formality as AI-like, and the stylometric signal reads low sentence variance
as AI-like. These users would receive an incorrect ⚠️ label. This is exactly why
the appeals workflow exists.

## Setup

1. Clone the repo and create a virtual environment:
```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
```

2. Install dependencies:
```bash
pip install -r requirements.txt
```

GROQ_API_KEY=your_key_here

4. Run the app:
```bash
python app.py
```

## Spec Reflection

**One way the spec helped during implementation:**
Writing out the three label variants in planning.md before touching any code
forced a real design decision upfront — what does a 0.5 confidence score mean
to a non-technical user? Having the exact label text written down made the
`get_label()` function trivial to implement and ensured the thresholds matched
the language. Without the spec, it would have been easy to just return a number
and figure out the labels later.

**One way implementation diverged from the spec:**
The spec described equal weighting between the two signals (50/50). During
testing it became clear that the stylometric signal is less reliable for short
informal posts — exactly the kind of content on a Love Island forum. The
weighting was adjusted to 60/40 in favor of the LLM signal, and planning.md
was updated to reflect this change.

## AI Usage

**Instance 1 — Flask app and Signal 1:**
I gave Claude the Detection Signals section and Architecture diagram from
planning.md and asked it to generate the Flask app skeleton, the LLM signal
function, and the audit log writer. It produced a working implementation. I
revised the LLM prompt to be more specific to Love Island fan content —
the original prompt was generic ("human or AI text") and I added explicit
examples of what real fan writing looks like vs AI output to improve scoring
accuracy.

**Instance 2 — Stylometric signal and confidence scoring:**
I gave Claude the Signal 2 spec and Confidence Scoring section from planning.md
and asked it to implement the stylometric function and weighted combination logic.
It produced the three-metric approach (sentence variance, TTR, punctuation
density). I adjusted the normalization ranges after testing — the original
sentence length variance normalization used std_dev / 4.0 which caused most
posts to score near 1.0. I changed it to std_dev / 8.0 to spread scores more
meaningfully across the range.
